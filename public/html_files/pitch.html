Font system: Pitch uses proprietary Eina01 (base) and Mark Pro (UI secondary) — both geometric sans-serifs common in startup design tools. Weight goes all the way to 800 (Extrabold) for display text, giving headlines extra visual authority. Letter-spacing spans negative display tightening (−1.6px) and positive caps tracking (+1.4px) — a total range of 3px.

Pitch has the most granular radius system in this series — 13 tokens from 3px to 56px (plus 50%). Also notable: two asymmetric tokens --radius-md: 3px 3px 0 0 and --radius-2xl: 8px 8px 0 0 (top-rounded only) — used for drawer handles and bottom-attached panels.
Border Widths
--border-width-hairline
1px · standard dividers, card borders (×69)
--border-width-thin
2px · active states, selected slides (×12)
--border-width-base
9px · thick selection ring for slide thumbnails (×16)
The 9px --border-width-base is the most unusual border token in this series — thick enough to be used as a visual "selection ring" around active slide thumbnails. Only Bloomberg (8px) comes close. This is a deliberate choice for a presentation app where slide selection state needs high visual emphasis.
Shadow / Elevation
Shadow Tokens
5 tokens — from flat to deep modal overlay
--shadow-sm
Dropdowns, tooltips (×24)
rgba(0,0,0,0.15) 0px 3px 10px — standard neutral drop shadow
--shadow-md
Cards, popovers (×2)
rgba(103,110,144,0.2) 0px 8px 26px — blue-tinted medium shadow matching the brand palette
--shadow-lg
Modals, dialogs (×2)
rgba(43,42,53,0.25) 0px 6px 27px — dark-indigo tinted, matches dark-gray base color
--shadow-xl
Full overlays, slide presenter mode (×2)
rgba(43,42,53,0.5) 0px 6px 27px — same base as LG but 50% opacity. The lg/xl pair share identical offsets, only the opacity differs — maximizing modal presence without changing shadow shape. Uniquely Pitch-matched: the shadow color is derived directly from the --color-dark-gray background, making shadows feel like the surface itself is casting a shadow.
Z-Index
LayerValueUsage
--z-1-1Behind content — slide background textures, canvas decorations
--z-21Base content — slide thumbnails, text blocks on canvas
--z-32Slightly elevated — selected elements, hover states, resize handles
--z-42,147,483,001Maximum (near int32 max, −646 offset) — presenter mode, full-screen overlays, critical modals that must sit above all embedded content and browser extensions
Pitch uses 2,147,483,001 — near int32 max (2,147,483,647) but with a −646 offset. This is the same family as Warp/Superhuman/xAI's int32 max strategy, but Pitch leaves room for exactly 646 values above it. This allows an internal "trump" value system where the presentation renderer can still guarantee stacking above the nav layer.

All 6 breakpoints start at 1082px minimum — Pitch is the only brand in this series with zero sub-1000px breakpoints. It's a pure desktop product; mobile web access to the editor is intentionally unsupported. The 1667/1670 near-pair is the same strategy as Warp's 809/809.98 — boundary max/min pairs that create clean layout transitions without rounding artifacts.

Pitch's opacity tokens are all in the upper range (50–85%) — none of the ultra-low ghost values found in most brands. These are used for disabled UI states (70%), focused panel overlays (80%), and tooltip/drawer backdrops (85%). The heavy opacity work is done via rgba() color values embedded in the color tokens directly.
